# Задание 1 — Аннотация `@Invoke`

### **Суть задачи**
Создать аннотацию для методов и обработчик, который автоматически находит и вызывает
все методы, помеченные этой аннотацией.

### **Как решено**
- Создана аннотация `@Invoke` с удержанием **RUNTIME**.
- В `ExampleMethods` два метода помечены `@Invoke`.
- `InvokeProcessor` через Reflection вызывает все такие методы.
- В `Main` есть пункт меню для проверки.
- В тестах проверяется:
  - метод найден (`find()`),
  - метод выполняется без исключений (`noException()`),
  - метод меняет состояние объекта (`changed()`).

---

# Задание 2 — Аннотация `@Default`

### **Суть задачи**
Аннотация должна хранить класс, который выступает «типом по умолчанию».

### **Как решено**
- Создана аннотация `@Default(Class value)`.
- `ExampleDefaultClass` помечен `@Default(String.class)`.
- `DefaultProcessor` выводит указанный класс.

---

# Задание 3 — Аннотация `@ToString`

### **Суть задачи**
Контролировать, какие поля включать в строковое представление объекта.

### **Как решено**
- Создан enum `Mode` (`YES`, `NO`).
- Аннотация `@ToString(Mode)`, по умолчанию `YES`.
- В `Person`:
  - `name` и `city` включаются,
  - `age` исключён (`Mode.NO`).
- `ToStringProcessor` формирует строку только из разрешённых полей.

---

# Задание 4 — Аннотация `@Validate`

### **Суть задачи**
Хранить массив классов, которые необходимо вывести или обработать.

### **Как решено**
- Аннотация `@Validate(Class[] value)`.
- `DemoClass` содержит список: `String`, `Integer`, `Double`.
- `ValidateProcessor` выводит каждый класс.

---

# Задание 5 — Аннотация `@Two`

### **Суть задачи**
Аннотация хранит два параметра: строку и число.

### **Как решено**
- Аннотация `@Two(first, second)`.
- `DemoTwoClass` задаёт значения.
- `TwoProcessor` выводит оба параметра.

---

# Задание 6 — Аннотация `@Cache`

### **Суть задачи**
Аннотация содержит массив строк — список областей кэширования.

### **Как решено**
- `@Cache(String[] value)` с пустым массивом по умолчанию.
- `CachedClass` задаёт 3 области: `"users"`, `"products"`, `"settings"`.
- `CacheProcessor` выводит все области или пишет, что список пуст.

# Тестирование (JUnit)

Реализован тест для аннотации `@Invoke`, который проверяет:

###  Поиск аннотированного метода  
Используется Reflection для нахождения метода, помеченного `@Invoke`.

###  Отсутствие исключений  
`assertDoesNotThrow` гарантирует безопасный вызов.

###  Побочный эффект  
Метод увеличивает счётчик — это проверяется в `changed()`.

###  Использование `@BeforeEach`  
Перед каждым тестом создаётся новый объект.
